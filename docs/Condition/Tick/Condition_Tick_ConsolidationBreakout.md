# 조건식 (Condition)

- STOM 주식 자동거래에 사용하기 위한 조건식 문서
- [[Back_Testing_Guideline_Tick]] 을(를) 기반으로 작성한 Tick 조건식
- [[Condition_Document_Template_Guideline]] 을(를) 바탕으로 템플릿 구조를 적용한 문서

## 개요

본 문서는 STOM 주식 자동거래 시스템에서 **횡보 돌파 전략**을 정의한다.

- **대상 시간 구간**: 09:05:00 ~ 14:30:00 (장중 전체)
- **대상 종목**: 관심종목, 가격대 1,000원~50,000원
- **전략 타입**: 횡보 돌파 (Consolidation Breakout)
- **핵심 변수**: 횡보구간범위, 최고현재가(300), 최저현재가(300), 초당거래대금평균(300), 돌파강도, 초당매수수량, 초당매도수량
- **업데이트 이력**:
  - 초기 문서 작성: 횡보 돌파 전략

## 소개

이 문서는 STOM 주식 자동거래 시스템에서 횡보 돌파 전략(Consolidation Breakout Strategy)을 위한 조건식을 제공합니다. 일정 기간 횡보 후 강한 돌파를 포착하여 추세 전환을 활용하는 전략입니다.

## 가이드라인

- 모든 조건식은 Python 스타일의 의사코드로 작성되어 있으며, STOM 백테스터가 인식하는 변수/함수 규칙을 따릅니다.

# Condition - Tick ConsolidationBreakout

## 전략 개요

횡보 돌파 전략은 일정 기간 횡보 후 강한 상승 돌파를 포착합니다:

- 최근 N틱 동안 횡보 구간 확인
- 고가 돌파 신호 포착
- 거래량 급증 동반 확인
- 돌파 강도 측정

## 조건식

### 매수 조건식

```python
# ================================
#  공통 계산 지표
# ================================
횡보구간범위          = ((최고현재가(300) - 최저현재가(300)) / 최저현재가(300)) * 100 if 데이터길이 > 300 else 100
횡보여부              = (1.0 <= 횡보구간범위 <= 3.0) if 데이터길이 > 300 else False
고가돌파여부          = (현재가 > 최고현재가(300)) if 데이터길이 > 300 else False
돌파강도              = ((현재가 - 최고현재가(300)) / 최고현재가(300)) * 100 if 데이터길이 > 300 else 0
거래량급증여부        = (초당거래대금 > 초당거래대금평균(300) * 2.0) if 데이터길이 > 300 else False
연속상승여부          = (현재가 > 현재가N(1) and 현재가N(1) > 현재가N(2))
매수압력강함여부      = (초당매수수량 > 초당매도수량 * 1.8)

매수 = True

# 공통 필터
if not (관심종목 == 1):
    매수 = False
elif not (90500 <= 시분초 <= 143000):
    매수 = False
elif not (1000 < 현재가 <= 50000):
    매수 = False
elif not (현재가 < VI아래5호가):
    매수 = False
elif 라운드피겨위5호가이내:
    매수 = False

# 횡보 확인
elif not 횡보여부:
    매수 = False

# 고가 돌파 확인
elif not 고가돌파여부:
    매수 = False

# 돌파 강도 확인
elif not (0.3 <= 돌파강도 <= 2.0):
    매수 = False

# 거래량 급증 확인
elif not 거래량급증여부:
    매수 = False

# 연속 상승 확인
elif not 연속상승여부:
    매수 = False

# 매수 압력 확인
elif not 매수압력강함여부:
    매수 = False

# 시가총액별 세부 조건
elif 시가총액 < 3000:
    if not (횡보구간범위 <= 2.5):
        매수 = False
    elif not (체결강도 >= 130):
        매수 = False
    elif not (회전율 > 2.0):
        매수 = False
    elif not (등락율각도(30) > 10):
        매수 = False
    elif not (현재가 > 이동평균(60)):
        매수 = False
    elif not (초당거래대금 > 초당거래대금평균(300) * 2.5):
        매수 = False
elif 시가총액 < 5000:
    if not (횡보구간범위 <= 2.0):
        매수 = False
    elif not (체결강도 >= 120):
        매수 = False
    elif not (회전율 > 1.5):
        매수 = False
    elif not (등락율각도(30) > 8):
        매수 = False
    elif not (현재가 > 이동평균(60)):
        매수 = False
    elif not (초당거래대금 > 초당거래대금평균(300) * 2.2):
        매수 = False
else:
    if not (횡보구간범위 <= 1.8):
        매수 = False
    elif not (체결강도 >= 110):
        매수 = False
    elif not (회전율 > 1.2):
        매수 = False
    elif not (등락율각도(30) > 6):
        매수 = False
    elif not (현재가 > 이동평균(60)):
        매수 = False
    elif not (초당거래대금 > 초당거래대금평균(300) * 2.0):
        매수 = False

if 매수:
    self.Buy(종목코드, 종목명, 매수수량, 현재가, 매도호가1, 매수호가1, 데이터길이)
```

## 조건식 개선 방향 연구

1) **횡보 구간 정확도 향상**
   - 다양한 시간 프레임 횡보 확인
   - 횡보 지속 시간 측정
   - 횡보 구간 신뢰도 평가

2) **돌파 신뢰도 평가**
   - 돌파 강도와 속도 측정
   - False breakout 필터링
   - 거래량 프로파일 분석

3) **횡보 구간 경계 판단**
   - 상단/하단 경계 명확화
   - 지지/저항 레벨 활용
   - 터치 횟수 카운팅

4) **돌파 후 추세 지속성**
   - 돌파 후 momentum 측정
   - 추가 상승 여력 평가
   - 재차 횡보 진입 탐지

5) **리스크 관리**
   - 돌파 실패 시 즉시 청산
   - 트레일링 스톱 적용
   - 시간 기반 청산 (15분)
