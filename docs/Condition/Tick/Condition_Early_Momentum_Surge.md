# 조건식 (Condition)

- STOM 주식 자동거래에 사용하기 위한 조건식 문서
- [[Back_Testing_Guideline_Tick]] 을(를) 기반으로 작성
- [[Condition_Document_Template_Guideline]] 을(를) 바탕으로 템플릿 구조를 적용한 문서

## 개요

본 문서는 STOM 주식 자동거래 시스템에서 **장초 모멘텀 급등 전략**을 정의한다.

- **대상 시간 구간**: 09:00:00 ~ 09:05:00 (장 시작 5분)
- **대상 종목**: 전 종목 대상
- **전략 타입**: 장초 급등 모멘텀 포착 초단타 전략
- **핵심 변수**: 초당거래대금, 체결강도, 급등 모멘텀
- **업데이트 이력**:
  - 초기 문서 작성: 장초 모멘텀 급등 전략

## 소개

이 문서는 장초 09:00:00~09:05:00 구간에서 급등 모멘텀을 포착하는 초단타 매매 전략입니다.
초당거래대금의 급격한 증가와 체결강도의 상승을 동반한 종목을 빠르게 진입하여 단기 수익을 실현합니다.

## 가이드라인

- 모든 조건식은 Python으로 작성되어 있으며, 주식 거래의 자동화를 목표로 합니다.
- 각 조건식은 특정 조건을 만족할 때 매수 또는 매도를 실행합니다.
- 최적화 조건식은 변수의 범위를 설정하여 최적의 매수/매도 시점을 찾습니다.
- 최적화 범위는 self.vars[정수] = [[시작값, 종료값, 간격], 최적값] 형식으로 설정합니다.

### 저장 이름 규칙

조건식과 관련된 다양한 테스트 및 최적화 조건을 저장할 때, 일관된 작명 규칙을 사용하여 관리의 용이성을 높일 수 있습니다.

1. **기본 규칙**:
   - `C`: Condition을 의미합니다.
   - `EMS`: Early Momentum Surge 를 의미합니다.
   - `B`: 매수 조건식을 의미합니다.
   - `S`: 매도 조건식을 의미합니다.
   - `BO`: 매수 최적화 조건식을 의미합니다.
   - `SO`: 매도 최적화 조건식을 의미합니다.

2. **예시**:
   - `C_EMS_B`: Early Momentum Surge 매수 조건식
   - `C_EMS_S`: Early Momentum Surge 매도 조건식
   - `C_EMS_BO`: Early Momentum Surge 매수 최적화 조건식
   - `C_EMS_SO`: Early Momentum Surge 매도 최적화 조건식

# Condition - Early Momentum Surge

## 조건식

### 매수 조건식

```python
# 장초 급등 모멘텀 포착 전략 (09:00:00 ~ 09:05:00)
초당순매수금액 = (초당매수수량 - 초당매도수량) * 현재가 / 1_000_000   # 단위: 백만원

# 시간 조건 (09:00:00 ~ 09:05:00)
if not (90000 <= 시분초 <= 90500):
    매수 = False

# 가격 및 등락율 조건
elif not (2000 < 현재가 <= 50000):
    매수 = False
elif not (2.0 < 등락율 <= 15.0):
    매수 = False

# VI 및 라운드피겨 제외
elif not (현재가 < VI아래5호가):
    매수 = False
elif 라운드피겨위5호가이내:
    매수 = False

# 거래대금 급증 조건
elif not (초당거래대금 > 초당거래대금N(1) * 1.5):
    매수 = False
elif not (초당거래대금 > 초당거래대금평균(30) * 2.5):
    매수 = False

# 체결강도 조건
elif not (체결강도 > 150):
    매수 = False
elif not (체결강도 > 체결강도N(1)):
    매수 = False

# 시가총액별 세분화 조건
elif 시가총액 < 2000:
    # 소형주: 더 강한 모멘텀 요구
    if not (초당순매수금액 > 80):
        매수 = False
    elif not (등락율각도(30) > 15):
        매수 = False
    elif not (최고초당매수수량(30) > 최고초당매도수량(30) * 3):
        매수 = False
    elif not (당일거래대금 > 30 * 100):
        매수 = False
    elif not (전일비 > 80):
        매수 = False
    elif not (매도총잔량 < 매수총잔량 * 2):
        매수 = False
    elif not (현재가 > 고가 - (고가 - 저가) * 15 / 100):
        매수 = False

elif 시가총액 < 5000:
    # 중형주
    if not (초당순매수금액 > 100):
        매수 = False
    elif not (등락율각도(30) > 12):
        매수 = False
    elif not (최고초당매수수량(30) > 최고초당매도수량(30) * 2):
        매수 = False
    elif not (당일거래대금 > 50 * 100):
        매수 = False
    elif not (전일비 > 60):
        매수 = False
    elif not (매도총잔량 < 매수총잔량 * 3):
        매수 = False
    elif not (현재가 > 고가 - (고가 - 저가) * 20 / 100):
        매수 = False

else:
    # 대형주
    if not (초당순매수금액 > 150):
        매수 = False
    elif not (등락율각도(30) > 10):
        매수 = False
    elif not (최고초당매수수량(30) > 최고초당매도수량(30) * 1.5):
        매수 = False
    elif not (당일거래대금 > 100 * 100):
        매수 = False
    elif not (전일비 > 50):
        매수 = False
    elif not (매도총잔량 < 매수총잔량 * 4):
        매수 = False
    elif not (현재가 > 고가 - (고가 - 저가) * 25 / 100):
        매수 = False

# 최종 매수 실행
if 매수:
    self.Buy(종목코드, 종목명, 매수수량, 현재가, 매도호가1, 매수호가1, 데이터길이)
```

### 매도 조건식

```python
# 단기 익절 및 손절 전략
if 등락율 > 29.5:
    매도 = True
elif 수익률 >= 2.0:
    매도 = True
elif 수익률 <= -3.0:
    매도 = True
elif 최고수익률 - 수익률 >= 1.5:
    매도 = True
elif 보유시간 > 120:  # 2분 초과 보유 시 청산
    매도 = True

# 시가총액별 세분화 매도 조건
elif 시가총액 < 2000:
    # 소형주: 빠른 청산
    if 0.5 <= 수익률 < 2.0 and (초당매도수량 - 초당매수수량) >= 매수총잔량 * 0.8:
        매도 = True
    elif 체결강도 < 100 and 체결강도N(1) > 120:
        매도 = True
    elif 등락율각도(30) < 5 and 등락율각도(30, 1) > 10:
        매도 = True

elif 시가총액 < 5000:
    # 중형주
    if 0.5 <= 수익률 < 2.0 and (초당매도수량 - 초당매수수량) >= 매수총잔량 * 0.6:
        매도 = True
    elif 체결강도 < 110 and 체결강도N(1) > 130:
        매도 = True
    elif 등락율각도(30) < 7 and 등락율각도(30, 1) > 12:
        매도 = True

else:
    # 대형주
    if 0.5 <= 수익률 < 2.0 and (초당매도수량 - 초당매수수량) >= 매수총잔량 * 0.5:
        매도 = True
    elif 체결강도 < 120 and 체결강도N(1) > 140:
        매도 = True
    elif 등락율각도(30) < 8 and 등락율각도(30, 1) > 13:
        매도 = True

if 매도:
    self.Sell(종목코드, 종목명, 매도수량, 현재가, 매도호가1, 매수호가1, 강제청산)
```

## 최적화 조건식

### 매수 최적화 조건식

```python
# 장초 급등 모멘텀 포착 전략 (최적화 버전)
초당순매수금액 = (초당매수수량 - 초당매도수량) * 현재가 / 1_000_000

# 시간 조건 (09:00:00 ~ 09:05:00)
if not (90000 <= 시분초 <= 90500):
    매수 = False

# 가격 및 등락율 조건 (최적화 변수)
elif not (2000 < 현재가 <= self.vars[7]):
    매수 = False
elif not (self.vars[8] < 등락율 <= self.vars[9]):
    매수 = False

# VI 및 라운드피겨 제외
elif not (현재가 < VI아래5호가):
    매수 = False
elif 라운드피겨위5호가이내:
    매수 = False

# 거래대금 급증 조건 (최적화 변수)
elif not (초당거래대금 > 초당거래대금N(1) * self.vars[10]):
    매수 = False
elif not (초당거래대금 > 초당거래대금평균(30) * self.vars[11]):
    매수 = False

# 체결강도 조건 (최적화 변수)
elif not (체결강도 > self.vars[12]):
    매수 = False
elif not (체결강도 > 체결강도N(1)):
    매수 = False

# 시가총액별 세분화 조건
elif 시가총액 < self.vars[13]:
    if not (초당순매수금액 > self.vars[14]):
        매수 = False
    elif not (등락율각도(30) > self.vars[15]):
        매수 = False
    elif not (최고초당매수수량(30) > 최고초당매도수량(30) * self.vars[16]):
        매수 = False

else:
    if not (초당순매수금액 > self.vars[17]):
        매수 = False
    elif not (등락율각도(30) > self.vars[18]):
        매수 = False
    elif not (최고초당매수수량(30) > 최고초당매도수량(30) * self.vars[19]):
        매수 = False

if 매수:
    self.Buy(종목코드, 종목명, 매수수량, 현재가, 매도호가1, 매수호가1, 데이터길이)
```

## 조건식 개선 방향 연구

### 1. 추가 지표 활용 연구

**우선순위: 높음**
- **호가창 두께 분석 추가**
  - 매수/매도 잔량 비율을 통한 지지/저항 강도 측정
  - 얇은 호가창 종목 필터링으로 급등락 리스크 감소
  - 구현 난이도: 낮음

- **거래대금 가속도 측정**
  - 1차 미분, 2차 미분 계산으로 거래량 가속 감지
  - 가속도가 양수일 때 매수 신호 강화
  - 구현 난이도: 중

### 2. 시간대별 전략 세분화 연구

**우선순위: 높음**
- **장 초반/중반/후반 조건 차등 적용**
  - 09:00-09:30: 갭 상승 전략 강화
  - 10:00-14:00: 모멘텀 추종 전략
  - 14:30-15:00: 단기 스캘핑 전략
  - 구현 난이도: 중

### 3. 매도 조건 고도화 연구

**우선순위: 높음**
- **동적 손절/익절 시스템**
  - 변동성 기반 손절폭 조정 (ATR 활용)
  - 트레일링 스톱 최적화
  - 구현 난이도: 중

- **부분 매도 전략**
  - 목표가 1차 도달 시 50% 매도
  - 나머지 50%는 트레일링 스톱 적용
  - 구현 난이도: 낮음

### 4. 리스크 관리 강화 연구

**우선순위: 높음**
- **허위 급등 필터링**
  - 급증 후 N초 지속 여부 확인
  - 일시적 스파이크 제거
  - 구현 난이도: 낮음

- **라운드 피겨 회피 강화**
  - 심리적 저항선 근처 진입 제한
  - 돌파 확인 후 진입
  - 구현 난이도: 낮음

### 5. 최적화 전략 개선 연구

**우선순위: 중간**
- **시가총액별 파라미터 세분화**
  - 초소형주 (1000억 미만) 별도 조건
  - 대형주 (1조 이상) 별도 조건
  - 구현 난이도: 중

- **백테스팅 기간 확대**
  - 다양한 시장 국면 반영
  - 과적합 방지
  - 구현 난이도: 낮음

### 구현 우선순위

1. **1단계 (즉시 구현)**:

   - 부분 매도 전략

   - 시간대별 전략 세분화

3. **3단계 (3개월 내)**:
   - 시가총액별 파라미터 세분화
